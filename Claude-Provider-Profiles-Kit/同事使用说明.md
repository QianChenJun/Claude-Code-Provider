# AI CLI Switcher 使用说明

这个压缩包用于在 Windows 上安装 `ccp` 和 `cdp` 两个命令：

- `ccp`：切换并启动 Claude Code 的不同供应商配置
- `cdp`：切换并启动 Codex CLI 的不同供应商配置

压缩包里不包含任何真实 API Key。安装后需要你自己填写供应商地址、模型和 API Key。

## 1. 安装前准备

至少先安装你要使用的 CLI：

```powershell
claude --version
codex --version
```

如果要使用 Web 管理页面，还需要 Node.js 18+：

```powershell
node --version
```

## 2. 解压并安装

把压缩包解压到任意目录，然后在 PowerShell 中进入解压目录：

```powershell
cd <解压目录>
```

建议先预检一次，不会写入文件：

```powershell
.\install.ps1 -DryRun
```

正式安装：

```powershell
.\install.ps1 -AddPath -Configure
```

安装完成后，重新打开 PowerShell 或 Windows Terminal。

如果系统提示脚本被阻止，可以先执行：

```powershell
Unblock-File .\install.ps1
```

再重新运行安装命令。

## 3. 配置供应商

安装时如果选择了配置向导，可以按提示填写。

也可以安装后手动运行：

```powershell
ccp setup
cdp setup
```

向导会询问：

- 配置 ID：例如 `mi`、`ds`、`openrouter`
- 显示名称
- 接口地址 `baseUrl`
- 默认模型 `model`
- API Key

API Key 默认写入 Windows 用户环境变量，不会写进配置文件。

## 4. 使用命令

查看配置：

```powershell
ccp list
cdp list
```

启动指定配置：

```powershell
ccp mi
cdp ds
```

不带参数可以打开交互菜单：

```powershell
ccp
cdp
```

打开 Web 管理页面：

```powershell
ccp manager
cdp manager
```

## 5. 常见问题

### 找不到 `ccp` 或 `cdp`

重新进入解压目录执行：

```powershell
.\install.ps1 -AddPath
```

然后重新打开 PowerShell / Windows Terminal。

### 提示缺少 API Key

如果刚刚通过 `ccp setup` 或 `cdp setup` 配置过 API Key，请重新打开终端。

### 新增配置后快捷命令不存在

执行同步：

```powershell
ccp sync
cdp sync
```

### 不想覆盖已有配置

默认不会覆盖已有配置文件。只有显式使用下面命令才会覆盖：

```powershell
.\install.ps1 -OverwriteConfig
```

## 6. 安全提醒

- 不要把 API Key 发给别人
- 不要把真实 API Key 写进压缩包、聊天记录或代码仓库
- 优先使用 `ccp setup` / `cdp setup` 配置密钥
- 如果需要手动编辑配置文件，优先使用 `apiKeyEnv`，不要使用明文 `apiKey`
